Riesgos y bloqueantes activos
Dashboard sin datos reales: Los stat cards muestran ceros hardcodeados. Las paginas internas (Alumnos, Grupos, Comunicados, Pagos, Documentos, Pickup) tienen nav pero no tienen implementacion. Para mostrar algo real a Habitat necesitamos al menos el CRUD de Alumnos y Grupos.
No hay mobile (iOS/Android): Las apps de padres no han iniciado. Si Habitat quiere ver el modulo de Pickup en el piloto, los padres necesitan la app. Este es el gap mas grande entre lo construido y lo prometido.
Stripe / CFDI sin integrar: El schema de pagos existe pero no hay logica de negocio. Si el piloto con Habitat incluye cobro de colegiaturas, esto bloquea el modulo de Pagos.
Sin CI/CD configurado: No hay GitHub Actions ni Vercel Preview Deployments documentados en el repo. El deploy es manual por ahora.
🎯
Foco recomendado para hoy — 15 de abril
1
Conectar datos reales al dashboard — queries a Supabase para stat cards: total alumnos, pagos pendientes, docs sin firmar. Sin datos reales el dashboard es decoracion.
2
Pagina de Alumnos — CRUD basico — Alta, edicion y listado de alumnos. Es el nucleo de todo lo demas (Pickup, Comunicados, Pagos dependen de alumnos).
3
Definir scope del piloto con Habitat — Que modulos va a usar Habitat en el piloto? Pickup, Comunicados, Pagos? Eso define el orden de desarrollo de las proximas 2 semanas.
4
Iniciar Pickup — logica de semaforo — Es el modulo diferenciador de Xokai y el de mayor complejidad tecnica (GPS + Realtime). Cuanto antes empiece, mejor.
